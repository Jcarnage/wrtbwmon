netMonitor is a companion set of scripts to the wrtbwmon program. It records all information into a sqlite database for easy processing and parsing as needed.

I developped these sets of scripts as I was looking for something more flexible than what was available out there. Piggy backing off the wrtbwmon application seemed like a natural fit as all the relevant data was already being collected and all I had to do was develop some scripts for a remote server (a PI 2 Model B in my case)

General Info

The application is set up to listen to port 5000 (UDP). If that port is taken by some other application in your setup, you can change that port by editing monitor.py and wrtbwmon.sh so that they are communicating on the same UDP port. Also, data is stored into the database every 15 minutes. Depending on your needs to can reduce or increase that timer by modifying the interval variable in monitor.py
